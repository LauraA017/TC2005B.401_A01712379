Laboratorio 1

Introducción a las aplicaciones web, HTML5 y ciclo de vida de los sistemas de información

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet: Es la red global de redes que conecta dispositivos en todo el mundo.

→ Internet es la red.

World Wide Web (WWW): Es un servicio que funciona sobre Internet y permite acceder a páginas web mediante navegadores usando HTTP/HTTPS.

→ La Web es uno de los servicios que usa esa red.

2. ¿Cuáles son las partes de un URL?

Una URL es una dirección web que se utiliza para identificar recursos en la web.

Ejemplo: https://www.ejemplo.com/blog/articulo?id=10

  • Protocolo: https
  • Dominio: www.ejemplo.com
  • Ruta: /blog/articulo
  • Parámetros: ?id=10

3. Propósito de los métodos HTTP

  • GET: Obtener información.
  • HEAD: Igual que GET, pero solo devuelve encabezados.
  • POST: Enviar datos para crear un recurso.
  • PUT: Reemplazar completamente un recurso existente.
  • PATCH: Modificar parcialmente un recurso.
  • DELETE: Eliminar un recurso.

4. Método HTTP para enviar formularios

POST

  • Envía los datos en el cuerpo de la petición.
  • No expone la información en la URL.
  • Es más seguro para datos sensibles (especialmente usando HTTPS).

5. Método HTTP al acceder a una página web

GET

Porque solo estás solicitando información al servidor.

6. Código HTTP 200

Significa “OK”.

La solicitud fue procesada con éxito y no ocurrió ningún error.

7. Error 404

Indica que el recurso solicitado no existe en la ubicación indicada.

Es responsabilidad del desarrollador cuando:

  • El recurso fue eliminado sin redirección.
  • Existe un enlace roto.
  • La estructura de rutas está mal configurada.

8. Error 500

Indica un fallo interno en el servidor.

  • Fallos en la lógica del backend.
  • Excepciones no controladas.
  • Errores en conexión a base de datos.
  • Problemas de configuración.

Es responsabilidad del desarrollador.

9. Elementos deprecated en HTML5

Un elemento deprecated:

  • Es obsoleto.
  • No se recomienda su uso.
  • Puede eliminarse en futuras versiones.

Elementos eliminados:

  • <font>
  • <center>
  • <tt>
  • <applet>
  • <dir>
  • <big>
  • <strike>

Atributos eliminados:

  • align
  • bgcolor
  • border
  • valign

10. Diferencias entre HTML4 y HTML5

Característica HTML4 HTML5
Año 1997 2014
Etiquetas semánticas No incluye modernas Incluye header, footer, nav, section, article, aside
Multimedia Requiere plugins Audio y video nativo
Formularios Básicos Nuevos tipos y validación nativa
Almacenamiento No disponible localStorage y sessionStorage
